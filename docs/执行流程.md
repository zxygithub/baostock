# BaoStock 数据下载器执行流程分析

## 项目概述

BaoStock 数据下载器是一个用于从 BaoStock API 下载中国A股市场数据（K线、财务数据、宏观经济数据等）并存储到 SQLite 数据库的工具。项目采用模块化设计，支持全量下载和增量更新。

## 项目结构

```
baostock/
├── config.yaml              # 下载配置（用户可修改）
├── pyproject.toml          # Python项目配置

│   ├── config.py           # 技术常量和字段定义
```

## 主要执行流程

### 1. 入口点：start.sh

`start.sh` 是项目的主要入口脚本，提供统一的命令行接口：

```bash
# 初始化数据库
./start.sh init

# 全量数据下载
./start.sh full

# 每日增量更新
./start.sh update

# 查看数据库状态
./start.sh status

# 查看日志
./start.sh logs

# 清理日志
./start.sh clean-logs [days]

# 清理临时表
./start.sh clean-tmp
```

### 2. 数据库初始化流程

```
start.sh init
    ↓
scripts/init_db.py
    ↓
src/db_manager.py.init_all_tables()
    ↓
创建所有数据表结构
```

**关键步骤：**
1. 创建 `data/` 目录（如果不存在）
2. 连接到 SQLite 数据库
3. 创建所有必要的表结构（约33个表）
4. 执行数据库迁移（如果需要）

### 3. 全量数据下载流程

```
start.sh full
    ↓
scripts/download_all.py
    ↓
分阶段下载所有数据
```

**下载阶段（按顺序执行）：**

#### 阶段1：数据库初始化
- 初始化所有表结构
- 执行数据库迁移

#### 阶段2：元数据下载（MetaDownloader）
- 下载股票基本信息（stock_basic 表）
- 下载交易日历（trade_cal 表）
- 下载行业分类（industry 表）

#### 阶段3：获取股票代码列表
- 从数据库读取所有A股股票代码
- 或从指定文件加载代码列表

#### 阶段4：宏观经济数据下载（MacroDownloader）
- 存款利率（deposit_rate）
- 贷款利率（loan_rate）
- 存款准备金率（rrr）
- 货币供应量（money_supply）

#### 阶段5：指数成分股下载（ComponentDownloader）
- 上证50成分股（sz50_stocks）
- 沪深300成分股（hs300_stocks）
- 中证500成分股（zz500_stocks）

#### 阶段6：指数K线下载（IndexDownloader）
- 上证指数（sh.000001）
- 深证成指（sz.399001）
- 创业板指（sz.399006）
- 上证50（sh.000016）
- 沪深300（sh.000300）
- 中证500（sh.000905）

#### 阶段7：股票K线下载（KlineDownloader）
- 日K线数据（all_stock_daily）
- 周K线数据（all_stock_weekly）
- 月K线数据（all_stock_monthly）
- 分钟K线数据（可选，默认关闭）

**K线数据特性：**
- 支持3种复权类型：adjustflags [1, 2, 3]
- 支持断点续传（checkpoint机制）
- 批量处理，支持Ctrl+C中断

#### 阶段8：财务数据下载（FinancialDownloader）
- 利润表（profit）
- 运营能力表（operation）
- 成长能力表（growth）
- 资产负债表（balance）
- 现金流量表（cashflow）
- 杜邦分析表（dupont）

#### 阶段9：公司报告下载（ReportDownloader）
- 业绩预告（forecast）
- 业绩快报（express）
- 分红送转（dividend）

#### 阶段10：分红数据下载（DividendDownloader）
- 分红信息（dividend）
- 除权除息信息（adjust_factor）

#### 阶段11：数据验证（可选）
- 检查数据完整性
- 生成数据库统计报告

### 4. 增量更新流程

```
start.sh update
    ↓
scripts/update_daily.py
    ↓
增量更新最新数据
```

**更新内容：**
1. 更新元数据（新增股票、交易日历等）
2. 下载最新的K线数据（最近N天）
3. 下载最新的财务数据（最新报告期）
4. 下载最新的公司报告
5. 下载最新的分红数据

### 5. 配置系统

项目采用**双配置文件**设计，职责清晰：

#### config.yaml（用户配置）

用户可以自由修改的运行参数，包括：
- API 限制（socket_timeout、daily_request_limit）
- 下载任务开关（enabled/disabled）
- 日期范围（start_date、start_year）
- 批处理参数（batch_size、batch_sleep）
- 股票筛选条件（filter）

#### src/config.py（技术常量）

开发者维护的技术常量，用户通常无需修改：
- 数据库路径（DB_PATH）
- K 线字段定义（DAILY_KLINE_FIELDS 等）
- 指数代码列表（INDEX_CODES）
- 内部参数（FINANCIAL_SLEEP、LOGIN_REFRESH_INTERVAL、MAX_RETRIES）

#### src/config_loader.py（配置加载器）

统一从 config.yaml 读取配置，提供便捷的访问函数：

```python
from src.config_loader import (
    load_config,              # 加载完整配置
    get_batch_size,           # 获取批处理大小
    get_batch_sleep,          # 获取批次休眠时间
    get_kline_start_date,     # 获取 K 线起始日期
    get_financial_start_year, # 获取财务起始年份
    get_reports_start_date,   # 获取报告起始日期
    get_index_kline_start_date, # 获取指数K线起始日期
    get_minute_frequencies,   # 获取分钟线频率列表
    is_download_enabled       # 检查下载开关
)
```

**配置加载流程**：
1. 所有下载器通过 `config_loader.py` 统一读取配置
2. 技术常量直接从 `config.py` 导入
3. 用户只需编辑 `config.yaml` 即可控制所有下载行为

### 6. 日志系统

- 所有操作都记录到 `logs/YYYYMMDD_HHMMSS.log` 文件
- 同时输出到控制台和日志文件
- 支持日志轮转和清理

### 7. 数据管理

```bash
# 查看所有表的数据量
./clean_data.sh --list

# 清理特定表
./clean_data.sh --table all_stock_daily

# 清理所有表（需要确认）
./clean_data.sh --all
```

### 8. 错误处理和恢复机制

1. **断点续传**：K线下载支持checkpoint，中断后可恢复
2. **信号处理**：支持Ctrl+C优雅中断
3. **超时机制**：网络请求设置超时，避免无限阻塞
4. **批量提交**：数据库操作批量提交，提高性能
5. **数据验证**：可选的数据完整性检查

### 9. 性能优化

1. **批量处理**：股票代码分批下载，每批200个
2. **数据库优化**：使用事务批量提交
3. **内存管理**：分页处理大数据集
4. **并发控制**：适当的休眠时间避免API限制

## 使用示例

### 完整数据下载
```bash
# 初始化数据库
./start.sh init

# 全量下载（跳过分钟数据和财务数据）
./start.sh full --skip-minute --skip-financial

# 查看下载状态
./start.sh status
```

### 日常维护
```bash
# 每日更新
./start.sh update

# 查看最新日志
./start.sh logs

# 清理30天前的日志
./start.sh clean-logs 30

# 清理临时表
./start.sh clean-tmp
```

## 注意事项

1. **API限制**：BaoStock API有调用频率限制，项目已内置休眠机制
2. **数据量**：全量下载数据量较大，需要足够的磁盘空间
3. **网络要求**：需要稳定的网络连接
4. **时间消耗**：全量下载可能需要数小时，建议在服务器上运行
5. **数据更新**：财务数据按季度更新，K线数据每日更新

## 扩展性

项目采用模块化设计，易于扩展：

1. **新增数据源**：继承 `BaseDownloader` 类实现新的下载器
2. **自定义配置**：修改 `config.yaml` 调整下载参数
3. **数据处理**：可在下载后添加数据清洗和转换逻辑
4. **输出格式**：可扩展支持其他数据库（MySQL、PostgreSQL等）
