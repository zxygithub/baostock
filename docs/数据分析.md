# BaoStock 数据库表 API 拉取前置条件分析

> 生成日期：2026-05-02
> 数据来源：`src/db_manager.py`、`src/config.py`、`config.yaml`、各 downloader 模块

## 一、全局前置条件（所有表共享）

| 条件 | 值/说明 | 来源 |
|---|---|---|
| **API 登录** | 每次下载前必须 `bs.login()` | `BaseDownloader.login()` |
| **每日请求上限** | 49,000 次/天，超限程序退出 | `config.yaml api.daily_request_limit` |
| **会话刷新间隔** | 30 分钟自动重新登录 | `config.py LOGIN_REFRESH_INTERVAL=1800` |
| **重试策略** | 最多 3 次，指数退避 | `config.py MAX_RETRIES=3` |
| **股票筛选条件** | `type=1 AND status=1`（仅上证A股且正常上市） | `config.yaml stocks.filter` |
| **批处理大小** | 每批 200 只股票 | `config.yaml stocks.batch_size` |
| **批次间休眠** | 2 秒 | `config.yaml stocks.batch_sleep` |
| **网络超时** | 30 秒 | `config.yaml api.socket_timeout` |

## 三、改进建议汇总

| 优先级 | 改进方向 | 影响表 | 预期效果 |
|---|---|---|---|
| 🔴 **高** | 财务表 6 张表可合并为一次请求（如果 API 支持） | profit/operation/growth/balance/cashflow/dupont | 减少 5/6 的请求量 |
| 🔴 **高** | 分红表遍历年份时可增量更新（只拉最新年） | dividend | 减少历史重复请求 |
| 🟡 **中** | K线复权类型可配置（不全量下载 3 种） | all_stock_daily/weekly/monthly | 减少 2/3 K线请求 |
| 🟡 **中** | 公司报告表可合并日期范围批量拉取 | performance_express/forecast_report | 减少按股票拆分的请求 |
| 🟢 **低** | 宏观数据表可拉长日期范围减少调用频次 | deposit_rate/loan_rate/reserve_ratio 等 | 微调优化 |
| 🟢 **低** | 分钟线默认关闭，已合理 | all_stock_5min/15min/30min/60min | 无需改进 |

## 四、请求量估算（全量下载场景）

假设条件：
- 有效股票数：约 5,000 只（type=1 AND status=1）
- 财务年份：2007-2026（20 年 × 4 季度 = 80 个季度）
- 分红年份：2007-2026（20 年）

| 数据类别 | 估算请求次数 | 占比 |
|---|---|---|
| K 线（日/周/月 × 3 复权） | 5,000 × 3 × 3 = **45,000** | ~85% |
| 财务数据（6 类 × 80 季度） | 5,000 × 6 × 80 = **2,400,000** | 远超日限，需多日完成 |
| 分红数据 | 5,000 × 20 = **100,000** | ~2% |
| 复权因子 | 5,000 = **5,000** | ~1% |
| 公司报告 | 5,000 × 2 = **10,000** | ~2% |
| 指数 K 线 | 8 × 3 = **24** | <1% |
| 宏观数据 | ~**50** | <1% |
| 元数据 | ~**10** | <1% |

> ⚠️ **结论**：财务数据全量下载远超每日 49,000 次限制，需分多日完成或优化请求策略。

## 五、数据下载依赖关系

```
stock_basic (元数据)
    ↓
all_stock (股票列表)
    ↓
├── all_stock_daily/weekly/monthly (K线)
├── dividend (分红)
├── adjust_factor (复权因子)
├── profit_data/operation_data/growth_data (财务)
├── balance_data/cash_flow_data/dupont_data (财务)
├── performance_express/forecast_report (报告)
└── sz50_stocks/hs300_stocks/zz500_stocks (成分股)

独立下载（无依赖）：
├── trade_dates (交易日历)
├── stock_industry (行业分类)
├── index_daily/weekly/monthly (指数K线)
└── deposit_rate/loan_rate/reserve_ratio/money_supply_* (宏观)
```
